Expand description

Pretty much equivalent to SourceId with a different serialization method.

The serialization for SourceId doesn’t do URL encode for parameters. In contrast, this type is aware of that whenever ResolveVersion allows us to do so (v4 or later).

We don’t care about the deserialization of this, as the url crate will always decode as the URL was encoded. Only when a Resolve turns into a EncodableResolve will it set the value accordingly via encodable_source_id.
